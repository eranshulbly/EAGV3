## Python Asyncio Best Practices

**1. Use `asyncio.run()` as Your Main Entry Point**
- Always use `asyncio.run()` to start your asyncio programs
- This function properly sets up and tears down the event loop
- It handles cleanup when exceptions occur

**2. Prefer Async Context Managers**
- Use async context managers (`async with`) for proper resource management
- Example: `async with aiohttp.ClientSession() as session:` for HTTP requests
- Ensures resources are properly acquired and released

**3. Always Await Coroutines**
- Never forget to await coroutines - a common mistake
- Without `await`, coroutines are created but never executed
- Use `await coroutine()` or `asyncio.run(coroutine())` to execute them

**4. Handle Cancellation Gracefully**
- Always catch `CancelledError` in tasks that might be cancelled
- Clean up resources in the exception handler
- Re-raise the error so the caller knows the task was cancelled
